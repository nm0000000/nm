# Лабораторная работа №1.3

### 1. Бинарная куча (Binary Heap)
**Python:**
Легко реализуется с помощью стандартной библиотеки heapq, которая основана на массиве и использует эффективный доступ O(log N) для вставки и извлечения минимального элемента.

**C++:**
Реализуется через стандартные контейнеры STL, такие как priority_queue, либо вручную через вектор и собственную логику перестройки кучи («heapify»).

**Java:**
Используется класс PriorityQueue, который построен на основе бинарной кучи и предоставляет эффективные операции вставки и выборки с временем O(log N).

### 2. Биноминальная куча (Binomial Heap)
**Python:**
Требует ручного управления памятью и списками для хранения ссылок на отдельные деревья, немного сложнее реализовать.

**C++:**
Эффективнее всего реализуем с помощью указателей и шаблонов классов. Сложность связана с необходимостью объединения отдельных деревьев и переорганизации структуры.

**Java:**
Необходимо создать классы для представления узлов и организации коллекции биноминальных деревьев. Поддерживать сложность операций непросто, но возможно.

### 3. Куча Фибоначчи (Fibonacci Heap)
**Python:**
Обычно требует ручной реализации сложной логики восстановления структуры после операций. Возможно применение модулей вроде queue, но полной замены функционала нет.

**C++:**
Одна из наиболее сложных реализаций, так как требуется поддержка большого набора внутренних операций, включая замедленное сокращение и отслеживание временных характеристик.

**Java:**
Написание требует детальной проработки классов и множественных изменений, включая объединение узлов и поддержку маркировки для задержки сокращений.

### 4. Хэш-таблица (Hash Table)
**Python:**
Стандартная библиотека предоставляет мощный словарь (dict), который легко использовать для эффективной работы с парами ключ-значение.

**C++:**
Используются контейнеры unordered_map или map, которые предоставляют быструю работу с элементами, обеспечивая среднюю временную сложность O(1).

**Java:**
Класс HashMap обеспечивает эффективную реализацию хэш-таблицы с удобной поддержкой пар ключ-значение и быстрой работой даже с большим количеством элементов.
