# Лабораторная работа 1.2
### Мультисписок
- **Python**: Используются вложенные списки, легко создаются и обрабатываются с помощью стандартных методов и циклов.
- **Java**: Применяется коллекция типа List (чаще всего ArrayList), содержащая внутренние списки.
- **C++**: Реализуется через стандартный контейнер std::vector<std::vector<T>>, поддерживающий динамическое изменение размеров.

### Очередь (FIFO)
- **Python**: Используется модуль queue.Queue, оптимизированный для многопоточности и безопасной работы с потоками.
- **Java**: Интерфейс Queue с популярной реализацией LinkedList, гарантирующей соблюдение правила FIFO.
- **C++**: Контейнер std::queue из стандартной библиотеки, работающий поверх другого контейнера (чаще всего deque или list).

### Дек
- **Python**: Модуль collections.deque, поддерживает быстрый доступ и модификацию как начала, так и конца очереди.
- **Java**: Интерфейс Deque с эффективной реализацией ArrayDeque, специально предназначенной для поддержки операций с двумя сторонами.
- **C++**: Стандартный контейнер std::deque, оптимизированный для быстрой вставки и удаления элементов с обоих концов.

### Приоритетная очередь
- **Python**: Библиотека heapq для построения двоичных куч или класс queue.PriorityQueue, поддерживающие естественный порядок элементов.
- **Java**: Класс PriorityQueue, автоматически сортирующий элементы по ключу (приоритету).
- **C++**: Контейнер std::priority_queue, использующий принципы адаптера и основанный на двоичной куче для эффективного извлечения максимального (или минимального) элемента.
